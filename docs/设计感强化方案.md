# Mio SLOWFIRE 设计感强化方案

> 基于现有框架（React + TailwindCSS + Zustand），在不引入重型设计库的前提下，用轻量方式提升整体质感。

---

| 资产 | 现状 | 评分 |

## 二、强化方向与优先级

### 🔴 最高优先级：动效体系（投入小，见效最快）

#### 1. 按钮按压反馈
所有可点击元素增加 `active:scale-[0.97]` + `transition-transform duration-150`：
```css
.press-feedback:active {
  transform: scale(0.97);
}
```

#### 2. 页面进入动画
目前只有 `animate-page-enter`（简单的 opacity + translateY）。可以按路由区分：
- 顾客端：从底部滑入（`translateY(20px) → 0`）
- 后台：淡入 + 轻微放大（`scale(0.98) → 1`）

#### 3. Toast 弹性动画
当前 Toast 是直接出现。改为：
```
从顶部滑入（translateY(-20px) → 0）
+ 弹性回弹（overshoot）
+ 3 秒后向上滑出消失
```

#### 4. 数字滚动效果
库存数量、订单统计等数字变化时，用 "odometer" 风格的滚动动画：
```
旧数字向上滑出，新数字从下方滑入
```

#### 5. 新订单实时提醒
后台收到新订单时：
- 卡片从右侧滑入（`translateX(100%) → 0`）
- 伴随轻微震动（`shake` 动画）
- 可选：提示音（简单的 "叮"）

---

### 🟠 高优先级：后台品牌一致性（弥合 C 端与后台的割裂）

#### 1. 后台背景升级
当前后台是纯白底。改为：
- 极淡的 paper-texture 背景（opacity 0.3）
- 或底部固定一个窑烤印章水印（opacity 0.05）

#### 2. 后台顶部品牌栏
当前 AdminHeader 比较素。增加：
- 左侧：火焰 SVG 图标 + "Mio 窑房" 手写体
- 右侧：当前批次号（如「第 3 窑」）

#### 3. 空状态品牌化
当前空状态是通用文字。改为品牌插图：
- 订单为空：面包师傅揉面团的 SVG + "面团还没醒好"
- 库存为空：空烤盘 SVG + "炉子还空着"
- 生产单为空：未点燃的窑 SVG + "还没封窑呢"

#### 4. 加载状态
当前 LoadingShell 是三个脉冲圆点。改为：
- 窑火跳动动画（SVG 火焰 shape-morphing）
- 或面包在烤炉中膨胀的动画

---

### 🟡 中优先级：数据可视化（让数字会说话）

#### 1. 概览页迷你趋势图
在统计卡片上增加 sparkline（迷你折线图）：
```
┌─────────────┐
│ 今日订单    │
│    23       │
│ ~~~~~/     │  ← 7 天趋势
└─────────────┘
```
不需要引入 Chart.js/Recharts，直接用 SVG path 画即可（轻量）。

#### 2. 生产完成进度条
生产单详情页增加环形进度条：
```
    生产中
   ╭─────╮
  │  ████ │  75%
   ╰─────╯
```

#### 3. 库存预警脉冲
库存 ≤ 2 时，卡片边缘有红色脉冲光环（`animate-pulse` + `ring-2 ring-red-400`）。

---

### 🟢 低优先级：暗色模式 + 字体升级（工作量大，后期再做）

#### 1. 暗色模式
后台长时间使用，暗色更护眼。设计方向：
- 背景：#1E1712（kiln）
- 卡片：#2A2018（比背景稍浅）
- 文字：#FAF6F0（ash）
- 强调色：#E84A2E（ember）不变
- 边框：#3D3028

Tailwind 实现：用 CSS variable 切换，或直接用 `dark:` 前缀。

#### 2. 中文字体升级
当前正文 fallback 到系统默认（macOS 是苹方，Windows 是微软雅黑），不同设备差异大。

引入方案：
- **霞鹜文楷**（开源，文艺感强，适合面包店的慢节奏气质）
- 加载方式：`@chinese-fonts/lxgwwenkai` 或 CDN
- 应用范围：正文、按钮、标签
- 保留 Inter 用于数字和英文

#### 3. 打印单字体内联
当前打印页依赖 Google Fonts CDN。如果打印机电脑没联网，字体 fallback 会导致排版错乱。

方案：
- 将 Google Fonts 的 CSS 内容内联到打印 HTML 中
- 或改用系统自带中文字体（SimSun/宋体）作为 fallback

## 三、实施建议

| 阶段 | 内容 | 预计工时 |
|------|------|----------|
| 第 1 期 | 动效体系（按钮反馈、Toast 弹性、数字滚动） | 2-3 小时 |
| 第 2 期 | 后台品牌感（背景纹理、空状态插图、加载动画） | 3-4 小时 |
| 第 3 期 | 数据可视化（sparkline、进度条、库存脉冲） | 3-4 小时 |
| 第 4 期 | 暗色模式 + 字体升级 | 4-6 小时 |

**建议先做第 1 期**，投入最小，用户对"手感"的提升感知最明显。

---

> 以上方案全部基于现有技术栈，不引入新依赖（除字体外），保持系统轻量。
