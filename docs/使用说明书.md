## 一、系统简介

Mio SLOWFIRE 是一款专为窑烤酸种面包店设计的在线订购与生产管理系统。它连接了顾客的线上预订、后厨的生产排程、以及门店的取货/配送管理，帮助小型烘焙工作室实现从「接单 → 生产 → 取货」的全链路数字化。

### 1.1 系统角色

| 角色 | 使用入口 | 主要职责 |
|------|----------|----------|
| **顾客** | 首页 `/` | 浏览商品、下单、查询订单、到店取货 |
| **门店运营** | 后台 `/admin` | 处理订单、管理库存、查看取货名单 |
| **烘焙师** | 后台 `/admin` → 生产标签 | 查看生产单、调整数量、确认出炉 |
| **总部管理员** | 后台 `/admin` | 商品管理、批次控制、门店设置、数据分析 |
| **收银员** | 收银台 `/admin/pos` | 现场点单、收款、打印小票 |

### 1.2 系统特点

- **批次销售模式**：以「窑次」为单位接单，封窑后统一生产，避免过度备料
- **多门店自提**：支持多个取货点，顾客可选最近门店
- **生产单数字化**：自动生成生产清单，支持数量微调、A4 打印、门店分拣
- **Canvas 窑烤凭证**：顾客下单后获得带「窑烤」印章的专属凭证，可长按保存
- **现场收银独立入口**：POS 模式不干扰线上订单流

## 二、顾客端使用指南

### 2.1 浏览商品

1. 打开首页，浏览当期上架的面包
2. 点击面包卡片可查看完整详情（配料、重量、价格、描述）
3. 分类标签可快速筛选：欧包/坚果、吐司、恰巴塔、贝果、软欧包
4. 售罄商品会显示灰色遮罩，无法加入购物车

> 💡 **提示**：商品展示采用「草间弥生」风格的圆点背景，不同品类有不同的配色主题。

### 2.2 下单流程

#### 第一步：添加商品
- 点击商品卡片上的「+」或数量按钮添加至购物车
- 购物车数量受实时库存限制，超出时会提示「库存不足」

#### 第二步：填写信息
点击「去结算」后：

1. **成为会员**：输入姓名 + 手机号，系统自动注册/登录，无需密码
2. **选择配送方式**：
   - **门店自提**（推荐）：选择就近取货门店，系统生成 6 位取货暗号
   - **本地配送**：填写详细地址，满 ¥150 免运费
   - **顺丰快递**：填写完整收件信息
3. **确认支付**：目前支持微信转账（可自定义付款说明）

#### 第三步：获得凭证
- 支付成功后弹出「窑烤凭证」—— 一张带有红色「窑烤」印章的 Canvas 绘图
- **手机端长按图片即可保存到相册**
- 凭证包含：订单号、取货暗号、商品清单、取货门店

### 2.3 查询订单

1. 首页点击「查询订单」
2. 输入**订单号**或**6 位取货暗号**
3. 查看订单状态：待确认 → 待生产 → 待自提 → 已完成
4. 也可通过「我的订单」侧边栏查看历史订单

### 2.4 批次开放状态

- **开窑期间**：正常下单，显示倒计时
- **封窑后**：只能浏览商品，不能下单，页面显示「本批次已截单」提示

## 三、管理后台使用指南

> 后台地址：`http://8.133.194.58/admin`  
> 登录账号：`1` / 密码：`1`

后台分为 **7 个标签页**：概览、生产、订单、会员、商品、库存、设置。

---

### 3.1 概览页

概览页是每日开工的第一站。

#### 指标卡片
顶部 6 张卡片实时展示：

| 指标 | 含义 |
|------|------|
| 今日订单 | 今天创建的所有订单数 |
| 待确认 | 顾客刚提交、尚未锁定的订单 |
| 待生产 | 已确认、等待入窑生产的订单 |
| 已出炉 | 面包已烤好、等待顾客取货/发货 |
| 已完成 | 已交货或已发货的订单 |
| 已取消 | 被取消的订单 |

#### 封窑 / 开窑（核心操作）

- **🟠 封窑**：当本期接单结束时点击。封窑后：
  - 所有「待确认」订单自动锁定为「待生产」
  - 系统根据全部订单自动生成一张「生产单」
  - 顾客端不能再下单
  - ⚠️ **封窑后无法恢复，请确认不再接单后再操作**

- **🟢 开窑**：当准备开始新一批次时点击。开窑后：
  - 顾客端恢复下单
  - 所有数据归为新批次

#### 取货名单

- 按取货门店分组展示所有待自提订单
- 每店显示：
  - 该店各商品汇总数量（方便提前备货）
  - 每位顾客的姓名、电话、取货暗号
- 可用于前台对照发面包

---

### 3.2 生产页

生产页是烘焙师的工作台。

#### 炉次生产看板

- 按照设置的炉次（如「第一炉 11:30」「第二炉 17:30」）分组
- 每个炉次卡片显示：该时段需烤的所有面包及数量
- 烘焙师按炉次依次生产，避免混乱

#### 配送队列

- 展示所有快递和同城配送订单
- 包含：收件人、电话、地址、金额
- 发货时可在此核对

#### 窑烤单记录

- 历史生产单列表，支持按日期筛选
- 每张生产单包含：生成时间、商品款数、总数量、当前状态
- 点击生产单进入详情页

#### 生产单详情（生产单打印页）

进入单张生产单后，可执行以下操作：

**1. 状态流转**
```
生产中 → 已出炉 → 已完成
```
- 点击对应按钮即可推进状态
- 「已完成」表示本单所有面包已交付/发出

**2. 编辑数量**
- 点击「编辑数量」按钮
- 在每个商品的数量输入框中修改实际数量
- 系统会自动按比例调整各门店的分配数量
- 修改场景：烤损、多烤、临时加单
- 修改后点击「保存」

**3. 打印 A4 制作单**
- 点击「打印 A4 生产单」
- 弹出紧凑排版的打印页面，包含：
  - 品类分组表格
  - 数量列（醒目红色）
  - 核对勾选框
  - 签名栏（和面/发酵、整形、出炉确认）
  - 红色「窑烤」印章

**4. 门店分拣单**
- 按门店展示该生产单中各店应分配的面包
- 每店可独立打印 A4 分拣单
- 分拣单包含：门店名、地址、取货码、面包清单、核对框

---

### 3.3 订单页

订单管理是运营的核心。

#### 筛选与搜索
- **搜索框**：支持订单号、顾客姓名、取货码模糊搜索
- **日期筛选**：按订单创建日期范围过滤
- **状态筛选**：快速查看某一状态的订单

#### 批量操作
1. 勾选左侧复选框选择多个订单
2. 点击顶部「批量操作」按钮
3. 可批量修改为：待生产、待自提、已完成、已取消

#### 导出 CSV
- 点击「导出」按钮，可将当前筛选条件下的订单导出为 CSV 表格
- 导出内容包含：订单号、时间、顾客、商品、金额、状态、门店等

#### 打印订单清单
- 点击「打印订单」，打开 A4 排版打印页
- 紧凑表格，适合贴在工作台旁核对

#### 单条订单操作
- 每条订单可展开查看详情
- 可单独修改状态、查看物流信息

---

### 3.4 会员页

- 展示所有注册会员列表
- 显示：姓名、手机号、订单数、消费总额、注册时间
- 支持搜索和导出 CSV

---

### 3.5 商品页

#### 总部权限
- **新增商品**：填写名称、分类、重量、价格、描述、上传图片
- **编辑商品**：修改任意字段，实时生效
- **上架/下架**：控制顾客端是否可见
- **推荐商品**：勾选后会在首页「热销排行」中展示

#### 新品上架流程
1. 在商品页点击「新增商品」
2. 填写信息并上传图片
3. 勾选「上架」
4. 到「库存」页为各门店分配初始库存
5. 到「设置」→「出炉批次」中将新品关联到对应炉次

#### 门店权限
- 仅可查看商品列表，不可编辑

---

### 3.6 库存页

#### 总部权限
- 为每个产品设置各门店的「分配数量」
- 实时显示：已售 / 总量、剩余数量
- 剩余 ≤ 2 时标红警告，提醒补货

#### 门店权限
- 只读查看本店库存余量

#### 库存计算逻辑
```
剩余库存 = 分配数量 - 已售数量
```
顾客下单时，系统根据选择的取货门店扣除对应库存。

---

### 3.7 设置页（仅总部可见）

设置页包含 5 个独立模块，每个模块需分别保存。

#### 批次设置
- **开放下单开关**：手动控制是否接受新订单
- **截单时间**：自动封窑时间（可留空）
- **出炉批次描述**：顾客端显示的批次文案
- **免运门槛**：满额免运金额（默认 ¥150）
- **基础运费**：不满免运门槛时的运费（默认 ¥10）

#### 出炉批次管理
- 增删改炉次（如「第一炉 11:30」「第二炉 17:30」）
- 为每个炉次勾选对应产品
- 生产看板将按此分组展示

#### 付款设置
- **微信号**：顾客转账时显示的收款人
- **付款说明**：转账时的备注提示
- **二维码图片**：上传收款二维码，顾客端可长按识别

#### 页面文案定制
可自定义顾客端各处文案：
- 结算页标题、副标题
- 购物车为空提示
- 封窑后提示语
- 下单成功标题与祝福语
- 页脚签名

#### 门店与账号
- 编辑门店名称、地址
- 控制门店是否开放自提
- 设置门店取货暗号前缀

### 3.8 现场收银台（POS）

> 地址：`http://8.133.194.58/admin/pos`

用于门店现场顾客直接购买，不经过线上预订流程。

#### 操作流程
1. 选择商品，点击加入收银单
2. 调整数量（加减按钮）
3. 选择收款方式：现金 / 微信转账 / 微信支付 / 支付宝 / 到店付
4. 点击「确认收款」
5. 订单自动标记为「已完成」，生成取货码

#### POS 订单特点
- 不占用线上库存（独立扣减逻辑）
- 不参与生产单汇总（已直接交付）
- 可在订单页查看，标记为 `[POS]`

## 四、核心工作流程

### 4.1 一日运营标准流程（SOP）

```
08:00  开窑 → 顾客可以下单
       └→ 管理员进入后台「概览」页确认状态

11:00  观察订单量，决定是否需要提前封窑
       └→ 生产页查看炉次看板，估算工作量

12:00  封窑 → 锁定所有订单，自动生成生产单
       └→ 进入生产单详情 → 打印 A4 制作单
       └→ 面包师按单生产，可微调数量

14:00  出炉 → 点击「已出炉」
       └→ 打印门店分拣单，按店分装

15:00  通知取货 → 顾客端状态变为「待自提」

18:00  关店核对 → 前台对照取货名单发面包
       └→ 所有订单完成后点击「已完成」
```

### 4.2 生产单数量调整场景

当实际生产与订单不符时，烘焙师可在生产单详情中修改数量：

| 场景 | 操作 | 影响 |
|------|------|------|
| 某款面包烤损 2 个 | 将数量减 2 | 各门店分配数按比例减少 |
| 临时多烤 5 个备用 | 将数量加 5 | 各门店分配数按比例增加 |
| 某店临时加单 | 直接修改该商品总数 | 系统自动重算各店分配 |

> ⚠️ 注意：修改数量仅影响生产单记录，不会自动修改顾客订单。如有缺货需联系顾客。

### 4.3 新批次启动流程

1. 确认上一批次所有订单已处理完毕
2. 进入后台「设置」→「批次设置」→ 确保文案已更新
3. 如有新品，到「商品」页上架，「库存」页分配，「设置」→「出炉批次」关联
4. 点击「开窑 · 新批次」
5. 顾客端恢复下单

## 五、常见问题 FAQ

**Q1：封窑后还能新增订单吗？**
> 不能。封窑后顾客端自动关闭下单入口。如需加单，只能走现场收银台（POS）。

**Q2：顾客没有保存窑烤凭证，还能取货吗？**
> 可以。顾客提供手机号或订单号，前台在「取货名单」中核对取货暗号即可发面包。

**Q3：生产单上的数量可以改吗？**
> 可以。进入生产单详情 → 点击「编辑数量」→ 修改后保存。系统会自动按比例调整各门店分配。

**Q4：为什么有些订单状态改不了？**
> 订单状态有固定流转规则：
> - 待确认 → 待生产（封窑时自动）
> - 待生产 → 待自提/待发货
> - 待自提 → 已完成（顾客取走后）
> - 待发货 → 已发货 → 已完成
> 
> 不能逆向修改，也不能跳过中间状态。

**Q5：如何新增一个取货门店？**
> 目前需联系技术管理员在数据库中新增。后台暂不支持自行添加门店（后续版本会支持）。

**Q6：库存不足时顾客还能下单吗？**
> 不能。顾客端会显示「售罄」或「仅剩 X 个」，超出部分无法加入购物车。

**Q7：POS 订单会影响线上库存吗？**
> POS 订单使用独立的库存扣减逻辑，不影响线上预订库存。但两者共享同一批物理面包，请运营人员注意统筹。

**Q8：如何导出某一天的订单？**
> 进入「订单」页 → 设置日期范围为当天 → 点击「导出 CSV」。

**Q9：生产单可以删除吗？**
> 目前不支持删除生产单。生产单作为生产记录永久留存，便于日后追溯和复盘。

**Q10：手机可以操作后台吗？**
> 可以。后台已做移动端适配，所有功能在手机上均可使用。但打印功能需要在连接打印机的电脑上操作。
